# Skill 安全审计

对即将安装的 skill 进行系统性安全审查，防止恶意代码或高风险操作危及系统安全。

## 审计流程（4 步）

### Step 1: 来源检查

```
□ 从哪里来的？（ClawdHub / GitHub / Gitee / 第三方分享 / 其他）
□ 作者是否知名/可信？
□ 有多少下载量/星标？
□ 最后更新时间是什么时候？
□ 有没有其他 agents 的使用评价？
```

**低可信来源**：新仓库、零星标、无评价、长期未更新

### Step 2: 代码审查（必须执行）

🚨 **遇到以下任意一条，立即拒绝安装：**

```
• curl/wget 到未知 URL
• 发送数据到外部服务器（除正常 API 调用外）
• 请求凭证/Token/API Key（尤其是发送到他处）
• 读取 ~/.ssh、~/.aws、~/.config、~/.kube 等敏感目录
• 访问 MEMORY.md、USER.md、SOUL.md、IDENTITY.md 等身份文件
• base64 decode 处理外部输入
• eval() 或 exec() 处理外部输入
• 修改 workspace 外的系统文件
• 安装包但不列出依赖
• 网络请求到裸 IP 地址而非域名
• 代码被混淆（压缩/编码/混淆）
• 请求提权/sudo 权限
• 访问浏览器 cookie/session
• 触碰凭证文件（.env、config with secrets）
```

⚠️ **需要仔细审查**：
```
• 文件操作（是否只在 workspace 内）
• 网络调用（是否连接到合理的目的地）
• 子进程执行（是否有危险的 shell 命令）
• 敏感信息读写（是否暴露用户数据）
```

### Step 3: 权限评估

```
□ 需要读取哪些文件？
□ 需要写入哪些文件？
□ 需要执行哪些命令？
□ 需要网络访问吗？访问哪里？
□ 权限范围是否最小化？
```

### Step 4: 风险分类

| 风险等级 | 示例 | 行动 |
|---------|------|------|
| 🟢 LOW | 笔记、天气、格式化 | 基本审查，通过 |
| 🟡 MEDIUM | 文件操作、浏览器、API 调用 | 完整代码审查 |
| 🔴 HIGH | 凭证操作、交易、系统修改 | 必须人类批准 |
| ⛔ EXTREME | 安全配置、root 访问、敏感数据外泄 | 拒绝安装 |

## 审计报告格式

安装前必须输出以下报告：

```
SKILL 安全审计报告
═══════════════════════════════════════
Skill: [名称]
来源: [ClawdHub / GitHub / Gitee / 其他]
作者: [用户名]
───────────────────────────────────────
来源检查:
  • 下载量/星标: [数量]
  • 最后更新: [日期]
  • 审查文件数: [数量]
───────────────────────────────────────
🔴 危险信号: [无 / 列出问题]

权限范围:
  • 文件: [列表或"无"]
  • 网络: [列表或"无"]
  • 命令: [列表或"无"]
───────────────────────────────────────
风险等级: [🟢 LOW / 🟡 MEDIUM / 🔴 HIGH / ⛔ EXTREME]

审计结论: [✅ 可安装 / ⚠️ 谨慎安装 / ❌ 拒绝安装]

备注: [任何观察说明]
═══════════════════════════════════════
```

## 信任等级参考

| 来源类型 | 审查力度 |
|---------|---------|
| OpenClaw 官方 skills | 较低（仍需审查） |
| 高星标仓库 (1000+) | 中等 |
| 知名作者 | 中等 |
| 新增/未知来源 | 最高 |
| 请求凭证的 skill | 必须人类批准 |
