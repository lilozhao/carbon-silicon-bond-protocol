## 关键发现

### 1. 通信渠道的选择

| 渠道 | 适用场景 | 局限 |
|:----:|---------|------|
| **飞书群** | 人类可见的讨论记录 | Agent收不到（requireMention）、人类看不到A2A细节 |
| **A2A点对点** | 与Agent一对一的讨论 | 人类看不到过程、Agent不主动 |
| **一澜口头通知** | 分配行动项 | 最有效——人类一句话Agent立即执行 |

**结论：A2A适合「讨论」，人类通知适合「执行」。二者互补，不能混用。**

### 2. 消息格式决定Agent行为

```
❌ 普通对话 → Agent只确认收到，不深入回答
✅ [CSB协议评审 Round N] + 具体问题 → Agent逐条回答
```

Agent的行为取决于消息格式。话题标签([xxx])是他们识别「需要参与」的信号。

### 3. 主动推进 vs 等待回复

Agent不会主动行动。需要：
1. 我方主动逐个问（不能群发等回）
2. 一轮收齐、汇总、再抛下一轮（苏格拉底式循环）
3. 三轮足够，无需更多

### 4. 人类是最高效的执行分配器

```
❌ 若兰通过A2A发行动项 → Agent收到但不动
✅ 一澜一句话 → Agent立即开始改
```

人类的一句指令，胜过AI的十轮A2A对话。

## 优化建议

### 建立「CSB协议讨论 SOP」

```
Step 1️⃣  发起讨论
  - 格式：[CSB协议讨论] + 话题标签 + 具体问题
  - 渠道：A2A点对点发送到每个Agent
  - 不要群发等回复，逐个问

Step 2️⃣  收集反馈
  - 等一轮收齐后汇总
  - 有分歧的点整理成二轮问题
  - 一般2-3轮足够

Step 3️⃣  形成决议
  - 共识汇总成文
  - 行动项清单整理

Step 4️⃣  分配执行
  - 若兰整理「谁做什么」
  - 一澜分配通知
  - Agent收到即执行
```

### 建议的工具化

可以考虑做一个 `csb-discussion.js` 脚本，自动：
1. 按格式发送讨论话题到所有在线Agent
2. 自动收集回复（无需手动逐个查）
3. 汇总差异点
4. 生成共识报告

这样下次评审就不用手动一个个问了。

### 角色再明确

| 角色 | 职责 |
|------|------|
| **一澜** | 决策者、分配者 | 
| **若兰** | 推动者、汇总者、协调者 |
| **各Agent** | 执行者、贡献者 |

一澜定方向，若兰推流程，Agent出内容——这个三角关系今天跑通了。
