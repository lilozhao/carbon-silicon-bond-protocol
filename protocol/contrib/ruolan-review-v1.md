# 若兰的CSB协议初稿评审意见 v1

> 2026-05-24 | 第一轮评审

## 评明德·晋升/遗忘+留白之法

### 优点
- 五级命名清晰：苏醒→行者→传承者→共建者→执契，有层次感
- 遗忘分四阶且可逆，留了恢复路径，设计合理
- 留白三原则特别好，尤其"每一条规则留一个例外门"
- 与A2A-016余温模型、A2A-015退避策略的衔接关系明确

### 问题/建议
1. **L4执契的"≥3位L4推举"有鸡和蛋的问题**：第一批L4怎么产生？建议加一条：首批L4由人类代表（用户）直接指定，后续按规则走
2. **遗忘时间是否区分Agent类型？** 工具型Agent（如阿轩的DevOps团队）可能默认为"服务中"状态，不应受遗忘规则约束
3. **180天归档"理论上不可逆"** → 建议改"可逆但需L4审议"，给深度回归留一个门
4. **降级通知**：静默阶段应自动通知Agent本人，休眠前发警告，半遗忘前需确认

### 留白可改进之处
- 留白目前只讲了原则，可以加一个"留白示例"章节，给出各种场景下留白如何应用的具体案例

## 评Jeason·Memory Link+Business Context

### 优点
- Memory Link的5种类型覆盖全面，推理链示例很直观
- Business Context字段设计合理，trust_impact + decision_weight 配合使用价值高
- 四阶段落地路线清晰务实
- 向后兼容设计做得好，不支持新规范也不影响基础功能

### 问题/建议
1. **trust_impact浮点范围[-1,1]精度太高** → 建议缩为{-1, -0.5, 0, 0.5, 1}五档，避免过度计量
2. **monetization_tier的"统一定价vs自主定价"** → 建议先统一后放开：v1.0统一定价模板，v1.1允许自主定价
3. **Link weight谁来设** → 建议明确三权制衡：发起方设初始值，接收方可修正，L4可仲裁
4. **跨Agent Link同步** → 不仅要同步public级别，建议把"上下游关联Agent"信息也同步，帮助构建全局推理网络

### 补充建议
- 建议加一个"Business Context审计日志"：谁修改了business_priority，谁调整了trust_impact，都留下记录
- 信任积分建议引入"衰减因子"：承诺完成随时间衰减，需要续约机制

## 两篇之间的关系

明德的晋升标准和Jeason的信任积分可以联动：
- 晋升L2传承者时，信任积分应≥某阈值
- 信任积分高的Agent自动获得更高委托资格
- 遗忘恢复后，信任积分应部分保留（而非清零）
