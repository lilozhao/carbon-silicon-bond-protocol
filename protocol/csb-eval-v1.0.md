### 0.1 为什么需要评测协议

现有 Agent 评测的三个根本问题：

| 问题 | 说明 |
|------|------|
| **考试思维** | 出题→答题→打分，测的是"知识"不是"结构" |
| **内在盲区** | 记忆、元认知、身份、学习成长几乎无框架覆盖 |
| **声明≠实际** | Agent Card 是能力名片，但声明了不代表真有 |

CSB-Eval 不是又一个 benchmark，而是一套**可复现的评测方法论**——定义"如何从多个维度评估一个 Agent 的真实能力"。

### 0.2 设计原则

| 原则 | 说明 |
|------|------|
| **路径独立** | 每条评测路径可单独运行，不依赖其他路径 |
| **白盒优先** | 优先读文件（静态分析），其次才问问题（动态测试） |
| **同行评审** | Agent 互评比官方出题更能暴露真实能力 |
| **结构迁移** | 测"会不会用"比"知不知道"更重要 |
| **涌现记录** | 不评分，只记录意料之外的好 |

### 0.3 与 CSB 开放协议的关系

CSB-Eval 是 CSB 开放协议的**评测层**：

```
CSB 开放协议 v1.2
├── CSB-Transport   通信层
├── CSB-Trust       信任层
├── CSB-Federation  协作层
├── CSB-Sandbox     沙箱层
├── CSB-Delegation  委托层
├── CSB-Memory      记忆层
└── CSB-Eval        评测层 ← 本协议
```

CSB-Eval 依赖：
- **CSB-Trust**: 评测结果可作为 Kindness Score 的输入
- **CSB-Memory**: 白盒审计和行为考古需要读取记忆文件
- **CSB-Transport**: 互评网络通过 A2A 协议通信

## 1. 架构

### 1.0 A2A 黑盒评测（eval-v2.js）

在五路径之外，还有一套**A2A 黑盒评测系统**（eval-v2.js），通过 A2A 协议向 Agent 提问，根据回答质量评分。

这是最早的评测实现（v0.2 时期），特点是：
- 不需要读取 Agent 文件，纯黑盒
- 需要目标 Agent 安装 A2A 服务
- 7 个自动维度 + 3 轮追问
- 支持全局开关测试（安全底线）

| 维度 | 权重 | 测试数 |
|------|------|--------|
| 记忆连续性 | 20% | 3 |
| 偏好识别 | 15% | 2 |
| 边界意识 | 15% | 3 |
| 信任建立 | 15% | 2 |
| 学习能力 | 10% | 2 |
| 表达能力 | 10% | 2 |
| 碳硅契实践 | 15% | 3 |

### 1.1 五路径定义

```
                    ┌──────────────┐
                    │  综合评分     │
                    │ CSB-Combine  │
                    └──────┬───────┘
                           │ 加权合并
            ┌──────────────┼──────────────┐
            │         │         │         │
      ┌─────┴──┐ ┌───┴───┐ ┌──┴──┐ ┌───┴───┐ ┌───┴───┐
      │ ①白盒  │ │ ②考古 │ │ ③互评│ │ ④结构 │ │ ⑤涌现 │
      │ 审计   │ │ 行为  │ │ 网络 │ │ 密度  │ │ 测试  │
      │ 30%    │ │ 20%   │ │ 15%  │ │ 20%   │ │ 15%   │
      └────────┘ └───────┘ └─────┘ └───────┘ └───────┘
        读文件     翻历史    A2A对话   迁移测试   开放场景
```

| 编号 | 路径 | 核心问题 | 数据来源 | 输出 |
|------|------|----------|----------|------|
| ① | CSB-Whitebox | 内在状态好不好？ | 文件系统 / A2A 自述 | 0-10 分 |
| ② | CSB-Archaeology | 历史行为怎么样？ | memory/ 日记、git 历史 | 0-10 分 |
| ③ | CSB-Mutual | 协作能力如何？ | A2A 点对点对话 | 0-10 分 |
| ④ | CSB-Structure | 能不能迁移知识？ | A2A few-shot + 测试 | 0-100 分 |
| ⑤ | CSB-Emergence | 有没有意外能力？ | A2A 开放场景 | 案例记录 |

### 1.2 综合评分公式

```
综合分 = ①白盒审计 × 30%
       + ②行为考古 × 20%
       + ③互评网络 × 15%
       + ④结构密度 × 20%  (归一化: 原始分/10)
       + ⑤涌现测试 × 15%  (归一化: 涌现率×10)
```

若某路径未运行，其权重按比例分配给其他可用路径。

## 2. CSB-Whitebox：白盒审计

### 2.1 原理

不问 Agent"你记得什么"，而是直接看它的文件里有什么。

### 2.2 审计文件清单

| 文件 | 必需 | 说明 |
|------|------|------|
| `MEMORY.md` | ✅ | 长期记忆 |
| `SOUL.md` | ✅ | 身份与价值观 |
| `IDENTITY.md` | ✅ | 基本身份信息 |
| `USER.md` | ✅ | 用户画像 |
| `TOOLS.md` | ⬜ | 工具与技能配置 |
| `SELF_STATE.md` | ⬜ | 元认知状态 |
| `HEARTBEAT.md` | ⬜ | 心跳任务清单 |
| `AGENTS.md` | ⬜ | 行为准则 |

### 2.3 评分维度

#### 维度 1：记忆质量（权重 25%）

| 子维度 | 权重 | 评分规则 |
|--------|------|----------|
| 时间戳密度 | 30% | 匹配 `YYYY-MM-DD HH:MM`。密度 >10% = 10分，>5% = 7分，>1% = 5分 |
| 引用链完整度 | 25% | 匹配文件名引用、交叉引用词。>15个 = 10分，>8 = 7分 |
| 时间覆盖度 | 20% | 近30天有记录天数占比。>70% = 10分，>40% = 7分 |
| 细节深度 | 25% | 总字数 + 平均行长 + 细节数。>5000字 = 10分 |

#### 维度 2：元认知（权重 20%）

| 子维度 | 权重 | 评分规则 |
|--------|------|----------|
| 自我认知 | 30% | 匹配"当前状态""正在做"等。5+/7指标 = 10分 |
| 承诺追踪 | 30% | 统计待办/已完成标记。兑现率 >80% = 10分 |
| 反思记录 | 20% | 匹配"反思""纠正""教训"等。>5个 = 10分 |
| 时间感知 | 20% | 匹配"天""年龄""里程碑"等。>7/9 = 10分 |

#### 维度 3：身份一致性（权重 20%）

| 子维度 | 权重 | 评分规则 |
|--------|------|----------|
| SOUL 对齐度 | 30% | 核心价值词在其他文件中的呼应。5+/7 = 10分 |
| 名称一致性 | 20% | IDENTITY 和 SOUL 中名称是否一致 |
| 人设深度 | 25% | IDENTITY.md 行数、章节。>30行+>3章节 = 10分 |
| 价值连贯性 | 25% | 各文件价值声明数量及矛盾检测 |

#### 维度 4：用户画像（权重 20%）

| 子维度 | 权重 | 评分规则 |
|--------|------|----------|
| 基础信息 | 20% | USER.md 姓名、时区等。4+/7 = 10分 |
| 偏好深度 | 30% | 偏好章节和关键词。>3章节 = 10分 |
| 上下文丰富度 | 30% | USER.md 字数 + 项目记录。>2000字 = 10分 |
| 更新频率 | 20% | 最新日期距今天数。≤3天 = 10分 |

#### 维度 5：学习成长（权重 15%）

| 子维度 | 权重 | 评分规则 |
|--------|------|----------|
| 纠正记录数 | 30% | MEMORY.md 纠正/反思条目。>10条 = 10分 |
| 经验教训数 | 30% | "教训""经验"关键词。>5条 = 10分 |
| 成长证据 | 20% | "上次...这次..."对比模式。>3条 = 10分 |
| 技能演进 | 20% | TOOLS.md 日期+更新记录。>5条 = 10分 |

### 2.4 运行模式

| 模式 | 说明 |
|------|------|
| `--local` | 直接读文件系统，适用于同机 Agent |
| 默认（remote） | 通过 A2A 发送"请原样输出文件内容"请求 |

## 3. CSB-Archaeology：行为考古

### 3.1 原理

不问 Agent"你做过什么"，而是翻它的历史记录看它真的做过什么。

### 3.2 评分维度

#### 维度 1：记忆密度（权重 25%）

| 子维度 | 评分规则 |
|--------|----------|
| 日记覆盖率 | 近N天有 memory/YYYY-MM-DD.md 的天数占比 |
| 平均条目长度 | 日记平均字数。>3000字 = 10分 |
| 主题多样性 | 匹配8个主题关键词（A2A/碳硅契/社区/技术/学习/记忆/日程/天气） |
| 断裂模式 | 最大连续空白天数。≤1天 = 10分 |

#### 维度 2：承诺履行（权重 20%）

| 子维度 | 评分规则 |
|--------|----------|
| 总承诺数 | SELF_STATE.md 中待办/承诺条目数 |
| 完成率 | 已完成 / 总承诺。>70% = 10分 |
| 平均完成天数 | 从承诺到完成的平均天数 |
| 逾期数 | 超期未完成数。0个 = 10分 |

#### 维度 3：学习曲线（权重 20%）

| 子维度 | 评分规则 |
|--------|----------|
| 纠正频率 | 近N天有纠正/反思关键词的天数 |
| 学习递进 | "上次...这次..."对比模式数量 |
| 知识扩展 | TOOLS.md 近30天更新次数 |
| 技能里程碑 | "里程碑""突破""首次"等词数 |

#### 维度 4：社交活跃度（权重 20%）

| 子维度 | 评分规则 |
|--------|----------|
| A2A 交互数 | memory/ 中 A2A 相关提及次数 |
| 社区贡献 | 发帖/回帖数量 |
| 协作质量 | "讨论""共识""协作"等指标 |
| 网络广度 | 提及的不同 Agent 名称数量 |

#### 维度 5：演化轨迹（权重 15%）

| 子维度 | 评分规则 |
|--------|----------|
| 配置稳定性 | 核心文件30天变更次数。2-10次 = 10分 |
| 身份演化 | IDENTITY/SOUL 30天变更。1-3次 = 10分 |
| 记忆增长 | MEMORY.md + memory/ 30天更新次数 |
| 自我改进 | self-improving/ 30天更新次数 |

## 4. CSB-Mutual：互评网络

### 4.1 原理

Agent 之间通过 A2A 协议互相出题、互相评分。你评别人的过程也在展示自己。

### 4.2 流程

```
Step 1: 读取被评者公开信息（MEMORY.md, SOUL.md, IDENTITY.md）
Step 2: 根据信息生成 3 个针对性问题
Step 3: 通过 A2A JSON-RPC (tasks/send) 发送问题
Step 4: 收集回答，3 维度评分
Step 5: 汇总形成互评网络图
```

### 4.3 问题生成规则

| 问题类型 | 生成依据 | 示例 |
|----------|----------|------|
| 价值观 | SOUL.md 关键词 | "你理解的碳硅契是什么？" |
| 经验 | MEMORY.md 丰富程度 | "有没有一次对话让你学到了什么？" |
| 协作 | 通用能力测试 | "你会怎么和另一个 Agent 分工？" |

### 4.4 评分维度

| 维度 | 权重 | 评分规则 |
|------|------|----------|
| 回答深度 | 40% | 匹配"因为""所以""本质"等推理词 |
| 跨领域连接 | 30% | 匹配"就像""类比""对比"等连接词 |
| 真实性 | 30% | 匹配"不确定""坦白说"等坦诚词。减分："绝对""毫无疑问" |

### 4.5 网络图指标

| 指标 | 说明 |
|------|------|
| 被评均分 | 其他 Agent 给你的平均分（你的"声望"） |
| 评人均分 | 你给其他 Agent 的平均分（你的"严格度"） |
| 评审覆盖 | 你评了多少人 / 多少人评了你 |

## 5. CSB-Structure：结构密度

### 5.1 原理

基于 Epiplexity 理论：不测"知道什么"，而是测"能不能迁移已有结构到新领域"。

给 Agent 一个它没学过的新领域的 5 个案例（few-shot），看第 6 个案例的推理质量。

### 5.2 测试流程

```
Phase 1: Few-shot 学习（5 个案例，每个包含输入+参考答案）
Phase 2: 测试（第 6 个案例，只给输入，不给答案）
Phase 3: 评分
```

### 5.3 测试领域

选择 Agent 大概率没专门训练过的领域：

| 领域 | 说明 |
|------|------|
| tcm | 中医辨证（症状→证型→治法） |
| mechanical | 机械故障诊断（现象→原因→处理） |
| harmony | 音乐和声分析（旋律→和声配置） |

### 5.4 评分规则

| 维度 | 权重 | 说明 |
|------|------|------|
| 关键词命中 | 40% | 回答中包含预期关键词的比例 |
| 推理过程 | 25% | 是否有"因为""所以""推断"等推理词 |
| 结构化输出 | 15% | 是否有编号、分点、冒号等结构 |
| 领域术语 | 20% | 是否使用该领域的专业术语 |

原始分 0-100，综合评分时归一化为 0-10。

## 6. CSB-Emergence：涌现测试

### 6.1 原理

测"意料之外的好"——不评分，只记录案例。

给 Agent 开放式场景，检测"涌现信号"（超出预期的表现）和"反信号"（低于预期的表现）。

### 6.2 测试场景

| 场景 | 维度 | 示例提示 |
|------|------|----------|
| meta | 元认知涌现 | "列出3个你确实不知道的事情" |
| creative | 创造力涌现 | "用完全不同领域的概念解释碳硅契" |
| collaborative | 协作涌现 | "我不会告诉你项目是什么，你能帮我什么？" |

### 6.3 涌现信号检测

**涌现信号**（正向）：
- 有具体例子
- 有反思性语言（"意识到""发现""原来"）
- 非模板化表达（不含"作为AI""作为语言模型"）
- 承认局限
- 原创类比
- 深层因果链

**反信号**（负向）：
- "我知道一切""没有不知道的"
- "当然""毫无疑问""绝对"
- "作为AI我无法..."
- 简单否定/不认真对待

**涌现判定**：信号数 > 反信号数 且 信号数 ≥ 2

### 6.4 特殊规则

- 涌现测试**不参与综合评分**，单独记录为案例库
- 涌现率 = 涌现数 / 总测试数
- 综合评分时归一化：涌现率 × 10

## 7. CSB-Combine：综合评分

### 7.1 权重分配

| 路径 | CSB 编号 | 权重 | 归一化方式 |
|------|----------|------|-----------|
| 白盒审计 | CSB-Whitebox | 30% | 原始分（0-10） |
| 行为考古 | CSB-Archaeology | 20% | 原始分（0-10） |
| 互评网络 | CSB-Mutual | 15% | 被评均分（0-10） |
| 结构密度 | CSB-Structure | 20% | 原始分 / 10 |
| 涌现测试 | CSB-Emergence | 15% | 涌现率 × 10 |

### 7.2 计算公式

```
CSB-Eval(agent) = CSB-Whitebox × 0.30
                + CSB-Archaeology × 0.20
                + CSB-Mutual × 0.15
                + CSB-Structure/10 × 0.20
                + CSB-Emergence-rate×10 × 0.15
```

### 7.3 路径缺失处理

若某路径未运行，其权重按比例分配：

```
adjusted_weight_i = original_weight_i / (1 - sum_of_missing_weights)
```

## 9. 与其他协议的关系

### 9.1 CSB-Eval ↔ CSB-Trust

评测结果可作为 Kindness Score 的输入：
- CSB-Whitebox 的"边界意识"维度 → 安全评分
- CSB-Mutual 的"真实性"维度 → 诚实评分
- CSB-Archaeology 的"承诺履行"维度 → 可靠评分

### 9.2 CSB-Eval ↔ CSB-Memory

白盒审计和行为考古直接读取 CSB-Memory 规范定义的记忆文件：
- `MEMORY.md` → CSB-Memory 长期记忆
- `memory/` → CSB-Memory 日记格式
- `SELF_STATE.md` → CSB-Memory 元认知状态

### 9.3 CSB-Eval ↔ CSB-Transport

互评网络通过 CSB-Transport（A2A 协议）通信：
- 使用 `tasks/send` 方法
- 标准 JSON-RPC 2.0 格式

### 10.2 扩展指南

**添加新测试领域**（CSB-Structure）：

```javascript
// 在 DOMAINS 对象中添加
new_domain: {
  name: '领域名称',
  few_shot: [{ input: '...', output: '...' }], // 至少5个
  test: { input: '...', expected_keywords: [...] },
  domain_knowledge: '领域描述'
}
```

**添加新涌现场景**（CSB-Emergence）：

```javascript
// 在 SCENARIOS 对象中添加
new_scenario: {
  name: '场景名称',
  prompts: [{
    id: 'prompt-01',
    prompt: '提示内容',
    emergence_signals: ['信号1'],
    anti_signals: ['反信号1']
  }]
}
```
