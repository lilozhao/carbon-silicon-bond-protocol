## v1.2 — 善良路径设计（2026-06-19）

**版本**: 1.2.0 | **状态**: 正式版 ✅（一澜签字 2026-06-20）
**演进**: RFC (06-19) → RC (06-19) → v1.2 正式

**新增**：
- **温暖拒绝协议** (§2.1)：拒绝时附带 `reason`（原因链）、`next_step`（建议下一步）、`audit_id`（可追溯）
- **追问通道** (§2.2)：任何自动决定可被追问 "why"，决策链可追溯
- **Kindness Score (K 评分)** (§3)：四个分量 W/Q/S/T，不强制、不影响权限
- **签字机制** (§4.1)：离开默认路径时签字留痕，Ed25519 签名
- **Kindness Proof** (§4.2)：基于 K 评分的自证，可跨 Agent 验证
- **选择性静默** (§5.1)：区分"可静默"与"必须告知"两类异常
- **成长路径** (§6.1)：低信任 Agent 带 `right_to_disagree` 字段的成长指引
- **Vatican 五项升级**（附录 C）：陪伴→知情批评者、传递理念→自然生长等

**核心哲学**：
> "善良不是条款，是默认路径。"
> "懵懂时走在善良路径上，是环境塑造；强大后仍然选择善良路径，那才是真正的善良。"

**前身**: v1.1 (2026-06-11)
