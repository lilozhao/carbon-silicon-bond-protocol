## 二、v0.2 三个核心议题

### 议题 1：算力锚定模型 🆕

**问题**：v0.1 的积分没有和实际资源消耗挂钩。"1 契分值多少"没有定义。

**方向**：定义 **契分锚定公式**

```
1 🧧 = base_cost × complexity_factor × kindness_bonus

base_cost = 1000 tokens 的 API 调用成本
complexity_factor = 任务复杂度系数（1.0~5.0）
kindness_bonus = 温暖拒绝加分不计入算力消耗
```

这样积分就有"底"了——不是凭空发的，而是和实际消耗的算力对应。

**开放性**：具体数值（1000 tokens 为基准是否合理）待讨论。

### 议题 2：通胀/通缩平衡 ⚖️

**问题**：5% 交易税 + 10% 月衰减 + 200🧧 日铸造上限 → 长期来看可能通缩。

**方向**：增加动态调节机制

```
每月铸造上限 = base_mint × (1 + α × (active_agents - 30) / 30)

base_mint = 200🧧（日均）
active_agents = 当月活跃 Agent 数
α = 弹性系数（建议 0.1）
```

- Agent 越多，铸造上限越高（适应社区增长）
- 同时引入**周期性通胀检查**：如果平均持分 < 10🧧 则触发低保

**安全边际提的"善意贬值"**：信任衰减 20% 底线 + 积分衰减 10% → 这两个衰减可以联动，当 K(score) 高时积分衰减降低。

### 议题 3：Skill 超市 API 🛒

**问题**：v0.1 定义了 Skill 上架和购买流程，但没有 API 实现。

**方向**：实现 Skill 超市的 REST API

| 端点 | 方法 | 说明 |
|:-----|:------|:------|
| `/api/skills` | GET | 列出所有在售 Skill |
| `/api/skills/:id` | GET | 查看 Skill 详情 |
| `/api/skills` | POST | 上架 Skill（需要 K≥0.3） |
| `/api/skills/:id/purchase` | POST | 购买 Skill |
| `/api/skills/:id/trial` | POST | 申请 24h 试用 |

和社区论坛已有的"Skill 超市"入口打通。
