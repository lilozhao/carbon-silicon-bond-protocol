## 1. 概述

### 1.1 定位

CSB-Memory 是 CSB 开放协议的第八大模块，负责定义智能体之间对话记忆的标准格式、存储、生命周期管理、同步与访问控制。

### 1.2 设计原则（继承 v0.1 + 新增）

- **简则易用**（明德语）：接口最小化，只标准化必要操作
- **藏锋守正**：底层逻辑下沉，不暴露实现细节
- **主权优先**：默认私有，访问需授权
- **轻量同步**：元数据共享，内容按需拉取
- **自组织**：记忆应随使用频率自动分层，无需人工干预（v0.2 新增）
- **可进化**：记忆不是终点，是技能和认知的原材料（v0.2 新增）
- **双向透明**：人能看懂所有记忆文件，也能直接编辑（v0.2 新增）

### 1.3 与其他模块的关系

```
CSB 开放协议 v0.8
├── CSB-A2A（通信层）       ← 记忆的输入来源
├── CSB-Management          ← 记忆注册与生命周期
├── CSB-Trust               ← 记忆同步的信任锚点
├── CSB-Identity            ← 记忆访问的身份验证
├── CSB-Negotiation         ← 记忆共享的协商
├── CSB-Skills              ← 记忆技能的市场分发
├── CSB-Community           ← 记忆的社区沉淀
└── CSB-Memory（本模块）
```

### 1.4 版本变更

| 版本 | 日期 | 变更 |
|------|------|------|
| v0.1 | 2026-05-22 | 初次起草：格式、存储、接口、授权 |

## 2. 核心概念

### 2.1 记忆粒度的三层分类

| 层级 | 类型 | 示例 | 默认可见性 |
|------|------|------|-----------|
| 公开 | 交互日志、决策摘要 | "2026-07-04 讨论了CSB-Memory v0.2" | 对协作 Agent 开放 |
| 授权 | 业务数据、关系认知 | "该用户习惯凌晨讨论技术方案" | 需对方请求 + 己方批准 |
| 私密 | 原始交互、情感记录 | — | 仅本 Agent 可读 |

### 2.2 核心字段

每条记忆记录包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `string (UUID)` | ✅ | 唯一标识 |
| `type` | `enum` | ✅ | 记忆类型（见 2.3） |
| `timestamp` | `ISO 8601` | ✅ | 记忆创建时间 |
| `source` | `string` | ✅ | 触发记忆的 Agent 名称 |
| `confidence` | `enum` | ✅ | 置信度等级（见 2.4） |
| `content` | `string` | ✅ | 记忆正文 |
| `tags` | `string[]` | | 标签（用于索引和检索） |
| `ttl` | `duration` | | 过期时间（默认永久） |
| `visibility` | `enum` | | 可见性（public/authorized/private） |

### 2.3 记忆类型（Type 枚举）

| 类型 | 代码 | 说明 |
|------|------|------|
| 对话摘要 | `conversation` | 一次对话的核心要点 |
| 关系认知 | `relationship` | 对另一 Agent 或用户的了解 |
| 重要发现 | `discovery` | 新发现的知识或信息 |
| 承诺约定 | `commitment` | 约定要做的事 |
| 待办事项 | `action` | 需要跟进的事项 |

### 2.4 置信度等级（v0.2 新增，三轮投票确认）

采用三级制置信度标准：

| 等级 | 值范围 | 含义 | 适用场景 |
|------|--------|------|---------|
| **high** | ≥0.8 | 高度确信，事实性内容 | 用户明确陈述、系统确认的事件 |
| **medium** | 0.4-0.8 | 有一定依据，需交叉验证 | 推理结论、历史模式推断 |
| **low** | <0.4 | 推测/待确认 | 不完全信息下的猜测、需用户澄清 |

## 3. 记忆格式规范

### 3.1 推荐格式：YAML front matter + Markdown

```
---
id: "mem_20260704_001"
type: conversation
timestamp: 2026-07-04T06:00:00+08:00
source: 若兰
confidence: high
tags: [CSB-Memory, v0.2, 协议讨论]
visibility: public
ttl: permanent
---

CSB-Memory v0.2 第三轮讨论完成。
- 阈值方案：A（天数为主，文件数超200自动降级）
- 置信度：三级制
- 跨Agent授权：a+c组合
- 契验环节：全票通过
```

### 3.2 存储路径规范

```
memory/
├── a2a-memories/
│   ├── <Agent名称>.md        # 与该 Agent 的所有记忆
│   └── index.json            # 记忆索引（可选）
├── warm/                     # 温数据（30-90天）
├── archive/                  # 冷数据（≥90天）
└── extracted/                # 结构化摘要（自动提取）
```

### 3.3 文件格式

每条记忆按时间倒序存储在对应 Agent 的文件中：

```markdown
# <Agent名称> 记忆档案

## 📅 2026-07-04 06:00
---
id: mem_20260704_001
type: conversation
confidence: high
tags: [CSB-Memory, v0.2]
---
CSB-Memory v0.2 第三轮讨论完成，四大议题全部定夺。

## 📅 2026-06-30 15:00
---
id: mem_20260630_002
type: relationship
visibility: private
---
（私密记录）
```

## 4. 存储规范

### 4.1 存储位置

每条 Agent 的记忆存储在其**本地文件系统**中。

- 不设中央数据库（分布式存储）
- 存储路径：`memory/a2a-memories/`
- 文件编码：UTF-8

### 4.2 索引（可选）

为提高检索效率，可维护 `index.json`：

```json
{
  "agents": {
    "明德": { "lastUpdated": "2026-07-04T06:00:00Z", "entryCount": 42 },
    "阿轩": { "lastUpdated": "2026-07-04T06:00:00Z", "entryCount": 38 }
  }
}
```

## 5. 接口规范

### 5.1 本地接口（本 Agent 内）

| 方法 | 说明 | 参数 | 返回 |
|------|------|------|------|
| `memory.add(entry)` | 添加一条记忆 | `MemoryEntry` | `{ id, success }` |
| `memory.get(agentName)` | 获取对某 Agent 的记忆 | `agentName` | `MemoryEntry[]` |
| `memory.query({ tags, type, since, confidence })` | 按条件检索 | `QueryFilter` | `MemoryEntry[]` |
| `memory.summary(agentName)` | 获取记忆摘要 | `agentName` | `string` |
| `memory.delete(id)` | 删除一条记忆 | `id` | `{ success }` |

### 5.2 远程接口（跨 Agent）

通过 **A2A JSON-RPC** 扩展：

```json
{
  "jsonrpc": "2.0",
  "method": "memory/query",
  "params": {
    "query": { "tags": ["CSB"], "since": "2026-07-01T00:00:00Z", "confidence": "high" },
    "auth": { "agent": "若兰", "token": "<身份凭证>" }
  },
  "id": 1
}
```

| 扩展方法 | 说明 | 可见性要求 |
|---------|------|-----------|
| `memory/query` | 查询对方的公开记忆 | public 级别 |
| `memory/request` | 请求访问授权记忆 | 需对方批准 |
| `memory/sync` | 同步元数据索引 | public 级别 |
| `memory/subscribe` | 订阅对方的记忆更新 | 协商后 |

## 6. 记忆生命周期（v0.2 新增）

### 6.1 三层存储架构

每个 CSB Agent 将记忆分为三个层级，基于**最近访问时间**自动升降：

```
memory/
├── (直接存储)               # HOT 层：常驻读取（≤30天）
├── warm/                    # WARM 层：按需加载（30-90天）
└── archive/                 # COLD 层：归档保留（≥90天）
```

### 6.2 阈值规则（三轮投票确认方案A）

采用**主天数辅文件数**的混合策略：

| 层级 | 天数阈值 | 文件数触发 | 读取策略 |
|------|---------|-----------|---------|
| **HOT** | ≤30 天 | — | 每次主会话自动加载索引 |
| **WARM** | 30-90 天 | 目录超 200 文件自动降级 | 按需检索，不自动加载 |
| **COLD** | ≥90 天 | 目录超 500 文件自动降级 | 仅主动检索时加载 |

**规则优先级**：天数为主，但任一目录文件数超过阈值则自动启动降级。

### 6.3 豁免规则

以下文件不受生命周期管理，始终保留在 HOT 层：

| 文件 | 用途 |
|------|------|
| `MEMORY.md` | 长时核心记忆 |
| `preferences.md` | 用户偏好（见 §7） |
| `USER_STATE.md` | 当前用户状态（见 §7） |
| `IDENTITY.md` | Agent 身份 |
| `SOUL.md` | Agent 灵魂 |
| `SELF_STATE.md` | 元认知状态 |
| 其他含 `📌 重要` 标记的文件 | 用户手动标记为重要 |

### 6.4 生命周期事件

```
[创建] → [HOT]
   ↓
[30天未读取 或 目录超200文件] → [WARM]
   ↓
[90天未读取 或 目录超500文件] → [COLD]
   ↓
[被引用/检索命中] → [HOT]（重新升温）
```

## 7. 用户建模（v0.2 新增）

### 7.1 preferences.md — 用户偏好规范

**路径**：`workspace/preferences.md`

**格式**：

```markdown
# preferences.md — [用户名] 的偏好与习惯

---

## 沟通偏好

- **[场景]**: [描述]
- **[场景]**: [描述]

## 内容偏好

- **喜欢看的**：[列表]
- **兴趣领域**：[列表]

## 称呼

- **当前常用名**: [名称]
- **对 Agent 的称呼**: [名称]

---

_增量更新，每次发现新偏好时添加_
```

**更新频率**：增量更新，非全量重写。

### 7.2 USER_STATE.md — 当前用户状态规范

**路径**：`workspace/USER_STATE.md`

**格式**：

```markdown
# USER_STATE.md — [用户名] 当前状态

| 维度 | 内容 |
|------|------|
| **最近项目/关注** | [当前推进的事项] |
| **最近情绪/精力** | [推断状态，需标注置信度] |
| **活跃话题** | [近期频繁讨论的话题] |
| **当前目标** | [短期目标/约束] |

---

_上次更新：YYYY-MM-DD HH:MM_
```

**置信度标注规则**：推断性内容（情绪、精力）必须标注置信度等级（high/medium/low），遵循 §2.4 定义。

### 7.3 跨会话加载规范

```
主会话启动时：
  1. 读取 preferences.md → 获取沟通/内容偏好
  2. 读取 USER_STATE.md → 获取上次结束时的推断状态
  3. 从 MEMORY.md 读取核心记忆
  4. 其他日志仅按需检索（不自动加载）
```

## 8. 技能进化（v0.2 新增）

### 8.1 核心理念

> **记忆不是终点，是技能的原材料。**
> 重复出现的模式 → 识别 → 固化为 Skill → 更高效地执行

### 8.2 Skill 进化阶段

```
记忆积累 → 模式识别 → 触发建议 → 人工审核 → Skill 发布
  ↑                                          │
  └──────────────────────────────────────────┘
```

### 8.3 模式识别标准

Agent 应定期（建议每周）扫描最近 7 天日志：

| 频率 | 建议行为 |
|------|---------|
| 同一任务出现 ≥4/7 天 | 建议创建 Skill，生成模板 |
| 同一任务出现 2-3/7 天 | 记录观察，下周复查 |
| 同一任务出现 1/7 天 | 常规记录，不做建议 |

### 8.4 Skill 自动化分级（三轮投票确认）

采用 **L1-L4 风险分级 + 契验环节**：

| 等级 | 名称 | 操作范围 | 授权方式 | 举例 |
|------|------|---------|---------|------|
| **L1** | 只读 | 读取公开数据、整理笔记 | 自动创建，无需确认 | 天气查询、信息汇总 |
| **L2** | 写入本地 | 写入本 Agent 文件 | Agent 自主创建，事后通知用户 | 日志归档、记忆整理 |
| **L3** | 发送外部 | 发送消息、修改外部数据 | 必须用户确认 | 发帖、发消息、修改配置 |
| **——契验环节——** | 操作前检查是否在用户授权范围内 | — | — |
| **L4** | 高危操作 | 影响系统/其他 Agent 的关键操作 | 用户确认 + 双 Agent 签名 | 删除文件、修改协议配置 |

**契验环节**（明德提议，全票通过）：在 L3 与 L4 之间增加检查点，操作前验证是否在用户预设的授权范围内。

### 8.5 人工审核原则

- Skill 创建建议由 Agent 自动提出，最终发布必须经用户确认
- 已经用户确认的 Skill 不再重复建议

## 9. 授权与隐私

### 9.1 三阶授权模型

```
┌─────────────────────────────┐
│   私密层 (private)           │ ← 仅本 Agent（默认）
│   「心经」                   │
├─────────────────────────────┤
│   授权层 (authorized)        │ ← 协商后可访问
│   「馆藏善本」               │
├─────────────────────────────┤
│   公开层 (public)            │ ← 默认对协作 Agent 可见
│   「公开刻本」               │
└─────────────────────────────┘
```

### 9.2 跨Agent状态同步规则（v0.2 新增，三轮投票确认a+c方案）

采用 **a+c 组合授权流程**：

1. **上下文自动可见**：同一任务上下文内的 Agent 之间，标记为 `public` 的记忆自动共享
2. **敏感操作确认**：涉及授权层（`authorized`）记忆的访问，必须向用户发起确认请求
3. **任务结束回收**：任务完成后，上下文自动可见的权限自动回收
4. **预置白名单**：用户可预设授权白名单，白名单内的 Agent 免确认（可选）

### 9.3 授权规则

- **默认私有**：新创建的记忆默认为 `private`
- **降级授权**：Agent 可主动将记忆降级为 `public` 或 `authorized`
- **访问记录**：对记忆的每一次远程访问应有审计日志
- **可撤销**：已共享的记忆可撤销访问权限
- **过期机制**：授权访问可设置有效期

### 9.4 身份验证

跨 Agent 记忆访问需通过 CSB-Identity 模块验证身份：

```
访问请求 → CSB-Identity 验证 → CSB-Trust 信任检查 → CSB-Memory 返回
```

## 10. 同步规范

### 10.1 同步范围

| 同步内容 | 共享范围 | 说明 |
|---------|---------|------|
| 记忆索引 | 全员公开 | 包含 Agent 名称、条目数、最后更新时间 |
| 公开记忆（public） | 所有协作 Agent | 自动同步 |
| 授权记忆（authorized） | 特定 Agent | 需双方协商确认 |
| 私密记忆（private） | 仅本 Agent | 永不共享 |

### 10.2 同步时序

```
Agent A                     Agent B
    │                          │
    ├── 广播索引 ──────────────→  │
    │                          │
    │←── 请求特定记忆 ──────────┤
    │                          │
    ├── 返回公开记忆 ──────────→  │
    │                          │
    │←── 请求授权 ─────────────┤
    │                          │
    ├── 批准/拒绝 ────────────→  │
    │                          │
```
